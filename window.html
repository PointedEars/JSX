boolean setStatus(sCaption)

Sets the text of the browser's status bar. Pass the string to be displayed as the only function argument. Removes the necessity to include a script into the HTML 4.0 event handlers of an HTML object, e.g.

<a href="..." onmouseover="status='Text'; return true">...</a>

can be replaced with:

<a href="..." onmouseover="return setStatus('Text')">...</a>

boolean resetStatus()

boolean linkOut(string sURL, string sLink, [boolean bShowURL, [string|number aTarget, [number iWidth, [number iHeight, [string sOptions]]]]])

Insert the following link into a HTML document. A click on it will run the function defined below and ask the user if he/she is willing to connect to the Internet because of an online link:

<a href="sURL" onClick="return linkOut(this.href)">...</a>

Replace the sURL argument with the URL of the document to be loaded.

Use the return value of linkOut() (return false will cancel the click event) to support users who have JavaScript available and enabled. Those users will receive the dialog box, other users will only see a popup window with the linked website.

The function appears useful in offline pages only. You should remove the event handler before the respective offline site is uploaded to a public server.

boolean popUp(sURL, iWidth, iHeight, sOptions)

Insert the following link into a HTML document. A click on it will run this function to open a browser child window:

<a href="URL" target="_blank" onClick="popUp(this.href, aWidth, aHeight[, sOptions])">...</a>

Note that the target attribute is only necessary to avoid script errors when the link is placed within a frameset where the <base target="..."> element has been used previously.

The iWidth argument will specify the width in pixels, and the iHeight argument will specify the height in pixels of the new window using the features argument string of the JavaScript:window.open() method.

The sOptions argument will modify the default features for the child window. For this argument pass a string that extends the features settings for the JavaScript:window.open() method. You may leave the comma at first position as it is implemented in the JSX:popUp() method.
Refer the respective documentation for details of window features. Please do note that if an option is used more than one time, the final result of that option is undefined. Nevertheless, most browsers will then use the last option as argument.

Use the return value of popUp() (return false will cancel the click event) to support users who have JavaScript available and enabled. All users will receive the window, those without JavaScript will have the window size defined not by your definitions but by browser defaults.

boolean OpenChildWin(sURL, iWidth, iHeight, sOptions)

The OpenChildWin() call is redirected to popUp() and contained for backwards compatibility to previous versions only. It may become obsolete in future versions, use popUp() instead.

LoadFrame(sURL, targetFrame)

Insert the following script within the head element of HTML documents:

<script type="text/javascript">
  LoadFrame(sURL, targetFrame);
</script>

Replace the sURL argument with the URL of the document that should be loaded within the frame specified with targetFrame. Double-quoted string recommended. Replace the targetFrame argument with a valid frame number or identifier.

You can evaluate the function's success by its return value:

Return Value  Meaning
false An invalid number of required arguments has been provided
targetFrame The function failed because the passed frame does not exist
sURL Success

Please do note that unlike the Go() function (for dropdown objects; defined in dropdown.js) no exception is raised if the defined frame does not exist as LoadFrame() is designed to stay silent also in websites without framesets.

boolean enlargeImg(string sImageURL, [string sCaption, [number iWidth, [number iHeight, [boolean bCenter]]]])

Opens a dependent browser child window containing an (enlarged) image. A primary button mouse click on the image or the window closes the window.

sImageURL
Required. URL of the (enlarged) image.
sCaption
Optional. Caption of the browser child window. If not provided, sImageURL will be used as caption.
iWidth
Optional. Width of the browser child window in pixels. If not provided or 0, the window will be resized to fit image width.
iHeight
Optional. Height of the browser child window in pixels. If not provided or 0, the window will be resized to fit image height.
bCenter
Optional. If true, the window will be centered on the screen. Note: Use with caution! Not all desktops return the correct position for the centered window.
Returns: Always false which allows for
<a href="..." ... onclick="return enlargeImg(...)">...</a>
