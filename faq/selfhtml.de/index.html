Vorwort

Dies ist ein informativer Text, der (technische) Fakten vermittelt, keine Meinungen!

Bei FAQs (Frequently Asked Questions = häufig gestellte Fragen) und bei Fragen, die darauf hindeuten, dass dem Teilnehmer der Unterschied zwischen Java und JavaScript nicht klar ist, sollte hierauf verwiesen werden.

Hilfreiche Kommentare und Verbesserungsvorschläge bitte nur per E-Mail an faq@PointedEars.de direkt an mich. Alles andere bitte nach /dev/tonne, danke.

Vielen Dank für die Idee zu diesem Posting an Dr. Hubert Partl, der einen ähnlichen Text regelmässig in den Usenet-Newsgroups de.comp.lang.javascript und de.comp.lang.java (hier mit einem Klick zum Abo) postet.

Worum geht es?

Viele Fragen, die gestellt werden, wurden vorher bereits diskutiert und beantwortet. Das erneute Stellen dieser Fragen nervt die Teilnehmer, die hier schon länger mitlesen und -posten (kurz: Regulars, von engl. "regular" = regelmässig), denn niemand möchte die gleiche Frage immer wieder beantworten.

Das hat dann oft gereizte Reaktionen zur Folge und der Fragende wundert sich darüber, denn er stellt ja die Frage aus seiner Perspektive das erste Mal.

Um Auseinandersetzungen zu vermeiden, mögen bitte die Regulars einfach auf dieses Dokument verweisen und die Frager versuchen, Probleme vor dem Posten über die Links in dieser FAQ sowie einschlägige Literatur und Selbststudium (einfach mal was ausprobieren!) zu lösen.

Weitere Information zum Auffinden von früheren Antworten findet man(n) auf der Website der Gruppe.

Ausserdem gibt es regelmässig Verwechslungen zwischen den beiden Programmiersprachen Java und JavaScript. Java ist nicht JavaScript (kurz: Java != JavaScript), und umgekehrt.

JavaScript

Script-Sprache z.B. innerhalb von HTML-Dokumenten.

ECMAScript-Implementation von Netscape (siehe Microsoft JScript).

Wird von Netscape Navigator bis 4.x (Mozilla/4.0) und Netscape Gecko (Rendering-Engine von Mozilla/5.0 -- d.h. Mozilla SeaMonkey, Mozilla Firefox/Thunderbird, Netscape 6+, Camino, ...) verwendet.
